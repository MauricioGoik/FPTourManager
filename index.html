C · Decisiones clave

⚡ Front navegable vs Backend

Priorizamos un front-end 100% funcional sobre conectar el servidor, aceptando que los datos no persisten si recargas la página.

📱 Diseño adaptable

Interfaz adaptable para móvil y escritorio, optimizada para que el uso sea ágil, claro y cómodo en cualquier dispositivo.

🧱 Monolito vs Microservicios

Backend futuro como monolito único para ganar rapidez de entrega. Microservicios se valorarán en la entrega final.

D · Riesgos y Mitigación

⚠

Fallo conexión Front ↔ API

En la demo final el servidor podría no estar disponible.

✓

JSON mockeado como "Plan B" siempre disponible en el código.

⚠

Tiempo perdido en diseño CSS

Estilizar desde cero consume tiempo sin aportar valor funcional.

✓

Tailwind CSS (componentes prefabricados). Sin CSS custom.

⚠

Resistencia de usuarios (Excel)

Profesores prefieren métodos ya conocidos al nuevo sistema.

✓

UX simple y móvil. Si es fácil, se adopta.
